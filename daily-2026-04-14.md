### 1. WordPress 插件供应链攻击：30款插件被植入后门
**排名：** #1 | 652 points | 180 comments  
**链接：** https://anchor.host/someone-bought-30-wordpress-plugins-and-planted-a-backdoor-in-all-of-them/

**是什么：** 一位攻击者通过 Flippa 平台收购了 30 款 WordPress 插件，然后在所有插件中植入了隐蔽后门代码。后门在 8 个月后才被激活，收购价格达 6 位数。

**为什么火：** 这是近年来最典型的供应链攻击案例之一。攻击者利用了 WordPress 生态的信任机制——用户信任知名插件，但忽视插件被收购后的安全性变化。事件涉及金额大、时间跨度长、手法极其隐蔽，引发了安全社区的高度关注。

**深度洞察：** Flippa 等二手插件市场的审核机制几乎为零。Widget Logic 插件事件只是冰山一角。这类攻击的可怕之处在于：攻击者可以等待很长时间（8个月）才激活后门，安全扫描工具在休眠状态下几乎无法检测到。这暴露了 WordPress 生态基础设施安全的系统性缺陷。

**产品机会：** 🔸 插件收购安全审计 SaaS——自动扫描被收购插件的代码变更  
🔸 WordPress 安全状态监控服务——实时追踪插件作者变更、代码差异  
🔸 信任评分系统——类似 npm audit，但面向 WordPress 插件生态

### 2. GitHub Stacked PRs：代码审查的范式革命
**排名：** #3 | 389 points | 235 comments  
**链接：** https://github.github.com/gh-stack/

**是什么：** GitHub 官方推出的 Stacked PRs 功能，允许将大型代码变更拆分为多个有序的小 PR，每个 PR 独立审查，最终一键合并全部。配合 `gh stack` CLI 使用。

**为什么火：** 大型代码库中，PR 体积过大是代码审查的最大痛点。一个 3000 行的 PR 让 reviewer 望而生畏，而 Stacked PRs 让每个变更层独立可审。GitHub 官方支持而非第三方工具，这意味着它将成为行业标准。

**深度洞察：** 这本质上是"增量式思维"在代码审查领域的落地。传统 PR 模型要求 reviewer 理解整个变更，而 Stacked PRs 让 reviewer 只需要理解每一层的增量。随着 AI 代码审查工具的普及，这种层级化的变更结构更容易被 AI 理解和审查。这是 GitHub 在 Copilot 之后最重要的产品迭代。

**产品机会：** 🔸 基于 Stacked PRs 的 AI 代码 reviewer——专注每层增量  
🔸 PR 依赖关系可视化工具——展示 stack 的有向无环图  
🔸 Stack 审查进度仪表盘——团队协作视角
